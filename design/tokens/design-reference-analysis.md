# Wall of Love 레퍼런스 분석

작성시각: 2026-03-14 14:24 KST
대상: `https://year-of-love.exploresyndrome.com/wall-of-love`
플랫폼: web

## 입력 이해

- 제품 타입: 감성형 전시/아카이브 레퍼런스
- 브랜드 캐릭터: 따뜻함, 장난기, 수집성, 에디토리얼
- 참고 자료: 사용자 제공 스크린샷 1장, 실제 웹 페이지
- 이번 산출물: 디자인 방향 분석 문서, 마스터 가이드, 작업 기록

## 가정과 누락

- 현재 프로젝트 `Ant Battle Log`는 코인 배틀 로그 제품이라 레퍼런스의 주제는 그대로 쓰지 않는다.
- 다만 "반복 카드 진열", "강한 수집감", "연한 배경 + 선명한 포인트", "아카이브형 편집" 문법은 일부 차용 가능하다고 본다.
- 프로젝트 기준 문서는 `docs/PRD.md`가 아니라 `docs/planning/01-prd.md`에 있었다.

## 관찰 결과

### 1. 구조

- 상단 소개 이후 매우 긴 세로형 카드 아카이브가 이어진다.
- 카드 폭은 좁고, 2열 반복이 강하게 보인다.
- 콘텐츠는 "날짜 / 컴포넌트명 / 작성자" 같은 짧은 메타 조합 중심이다.

### 2. 시각 시스템

- 배경색은 크림 계열 `#FFF8ED`에 가깝다.
- 카드 베이스는 핑크 계열 `#FFC7E9`가 중심이다.
- 포인트는 레드 `#FD1212`, `#FF0000`, 보조 하이라이트는 옐로 `#FFED91`가 보인다.
- 컬러를 많이 쓰지만 전체 톤은 여전히 부드럽다.

### 3. 타이포그래피

- 장식적인 대문자 반복과 짧은 카드 라벨이 핵심이다.
- 긴 본문보다는 제목 반복, 카드 이름, 날짜처럼 짧은 텍스트를 리듬감 있게 배치한다.
- 읽는 경험보다 "둘러보는 경험"을 우선한다.

### 4. 감정과 브랜딩

- 디지털 제품 소개 페이지라기보다 사랑 기록 전시장에 가깝다.
- 구조적으로는 그리드지만 정서적으로는 스크랩북에 가깝다.
- 정돈된 규칙성과 장난스러운 색 사용이 동시에 존재한다.

### 5. 현재 프로젝트에 가져갈 수 있는 점

- 캐릭터 카드나 배틀 기록 카드에 수집형 진열 문법을 적용할 수 있다.
- 동일 컴포넌트의 반복 노출이 많을 때, 카드 크기와 간격을 강한 규칙으로 통일하면 인상이 선다.
- 배경은 따뜻하게 두고, 핵심 상태값만 강한 포인트 컬러로 주는 방식은 모바일에서도 유효하다.

### 6. 그대로 가져오면 안 되는 점

- 현재 프로젝트는 데이터 이해와 선택이 중요해서 이 정도 밀도의 장식성은 메인 플로우를 방해할 수 있다.
- 과한 핑크/레드 중심 팔레트는 코인 배틀 제품의 신뢰감을 떨어뜨릴 수 있다.
- 긴 세로 아카이브 구조는 홈이나 캐릭터 도감에는 맞아도 배틀 진행 화면에는 맞지 않는다.

## 적용 권장안

- 메인 배틀 화면에는 쓰지 말고 `캐릭터 도감`, `전적 히스토리`, `결과 아카이브` 화면에 제한적으로 차용한다.
- 컬러는 이 레퍼런스보다 더 절제해서 크림 배경 + 포인트 레드 1종 + 서브 컬러 1종 정도로 줄이는 게 낫다.
- 카드 반복 리듬은 유지하되, 정보 우선순위는 지금 프로젝트 맥락에 맞게 더 명확하게 재정렬해야 한다.
