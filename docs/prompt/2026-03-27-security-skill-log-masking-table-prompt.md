# 보안 스킬에 로그 마스킹 규칙 표 추가 프롬프트

- 작성시각: 2026-03-27 12:13:38 +09:00
- 프롬프트:
  - 로그에 남기면 안 되는 필드 마스킹 규칙 자체를 표로 만들어서 email, phone, card, jwt, api key 각각 어떤 형태로 redaction해야 하는지까지 스킬에 넣어. 그리고 fireauto-secure 스킬 실행했을떄 실제로 모든 항목 다 검증하는지 테스트 한후 결과 알려줘
- 해결하고자 한 문제:
  - 민감 로그 필드별 마스킹 기준을 명문화하고, 스킬 반영 여부를 실제 검증하는 것
- 해결된 것:
  - 마스킹 규칙 표를 스킬 reference에 추가함
  - 요청한 모든 민감 필드가 스킬에 반영됐는지 커버리지 체크를 수행함
  - validator와 grep 테스트 결과를 확인함
- 아직 해결되지 않은 것:
  - 스킬은 실행형이 아니라 문서형이라, 실제 검증은 구조/커버리지 테스트로 해석해 수행함
