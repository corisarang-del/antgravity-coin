# 디자인 토큰 기준 페이지 전수 점검 및 UI 정렬

- 작성시각: 2026-03-15 21:49:19 +09:00
- 해결하고자 한 문제:
  - `bootstrap-app/design/design-tokens.md` 기준으로 현재 모든 페이지가 토큰 규칙과 얼마나 맞는지 확인하고 싶었음.
  - 크기는 유지하면서 레이아웃, UX/UI, 색감, 간격이 어긋난 부분만 정리해야 했음.
- 사용한 프롬프트/지시:
  - 디자인 토큰 문서와 실제 페이지를 비교해 토큰 위반이나 일관성 저하가 있는 부분을 찾는다.
  - 홈, 캐릭터 도감, 배틀 선택/대기/결과 화면을 실제 브라우저로 확인한다.
  - 크기 변경 없이 색감, 표면 레이어, 포커스/선택 상태, 공통 레이아웃 흐름만 수정한다.
  - `pnpm test`, `pnpm lint`, `pnpm typecheck`로 검증한다.
- 해결된 것:
  - 홈/도감 카드와 검색 패널에 토큰 기준 표면 레이어와 포커스 상태를 보강함.
  - `pick/result/waiting` 페이지에 공통 헤더와 카드형 빈 상태를 추가해 흐름을 통일함.
  - 선택 버튼과 시간 선택기의 상태 표현을 토큰에 맞게 정리함.
  - 브라우저로 `/home`, `/characters`, `/battle/[coinId]/pick`, `/battle/[coinId]/waiting`, `/battle/[coinId]/result`를 확인함.
- 해결되지 않은 것:
  - 결과 페이지의 실제 승패 렌더링 상태는 배틀 스냅샷 데이터가 없는 흐름이라 브라우저에서 최종 카드까지는 확인하지 못했음.
