# research 구조 보강 프롬프트

- 작성시각: 2026-03-17 03:25 KST
- 해결하고자 한 문제: 프로젝트 구조를 실제 구현 기준으로 다시 파악하고, `research.md`에는 기존에 없는 내용만 추가해야 했다.

## 사용한 프롬프트

```text
프로젝트 구조 완벽히 이해하고 research.md에 기록. 단 이미 기존내용에 없는 부분만 추가
```

## 해결된 것

- 사용자 요청을 기준으로 루트 `research.md`만 대상으로 삼았다.
- 먼저 기존 `research.md` 내용을 확인한 뒤, 중복 없는 정보만 선별해 추가했다.
- 코드베이스의 실제 디렉터리 구조, 운영 API, 파일 저장소, 런타임 provider 라우팅, 테스트 배치 정보를 추가했다.

## 아직 안 된 것

- 구조 확인을 위해 핵심 파일 위주로 읽었고, 모든 파일을 전수 검토하지는 않았다.
- 보조 작업공간 폴더의 생성 목적까지는 별도 인터뷰나 히스토리 확인 없이 관찰 수준으로만 정리했다.
