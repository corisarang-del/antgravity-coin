# 6단계 차트형 타임프레임 전환 프롬프트

작성시각: 2026-03-20 11:15 KST

## 프롬프트 요약

타입과 저장 스키마를 6단계로 확장하고, `TimeframeSelector`와 배틀 UI를 `5분 후`, `30분 후`, `1시간 후`, `4시간 후`, `24시간 후`, `7일 후` 기준으로 바꾸고, `resolveBattle`를 단계별 결과 계산 구조로 리팩토링하고, 문구도 실시간 차트형 결과 톤으로 정리하기.

## 해결하려고 한 문제

- 두 단계 타임프레임의 한계
- 결과 계산 로직의 낮은 확장성
- 화면 카피와 사용자 기대의 불일치
- 저장 타입 문서와 실제 구현의 드리프트 가능성

## 해결된 것

- 공통 타임프레임 타입 추가
- 결과 계산 구조 개편
- 선택, 대기, 결과 UI 카피 정리
- 문서와 테스트까지 함께 갱신

## 아직 안 된 것

- 실제 캔들 히스토리 연동은 아직 아니다.
