# 서버 배틀 snapshot 저장 프롬프트

작성시각: 2026-03-20 14:25 KST

## 프롬프트 요약

배틀 완료 직후 토론 snapshot 자체를 서버에 저장하고, 이후 결과 정산 시 로컬 snapshot이 사라져도 서버 snapshot으로 리포트와 결과를 복구할 수 있게 만들기.

## 해결하려고 한 문제

- localStorage 의존도가 너무 높아서 장기 정산 흐름에 취약한 점
- 결과 화면에서 snapshot 메시지 부재로 outcome 생성이 막히는 점

## 해결된 것

- 서버 snapshot 저장소, API, 클라이언트 저장 흐름, 결과 정산 fallback 경로를 연결했다.

## 아직 안 된 것

- 사용자 계정 단위의 영구 복원 정책까지는 아직 없다.
