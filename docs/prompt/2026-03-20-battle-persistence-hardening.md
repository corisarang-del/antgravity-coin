# 배틀 저장 안정성 강화 프롬프트

작성시각: 2026-03-20 00:00 KST

## 프롬프트 요약

배틀 결과 저장 전에 문자열 정합성 검증을 강화해서 깨진 텍스트를 영속화하지 않기, 파일 저장소 write를 최소한 battle 단위 직렬화로 보호하기, `/api/battle/outcome`와 적용 마킹을 서버 쪽 idempotent 트랜잭션처럼 합치기, `battleSnapshot`과 `userBattle`에 TTL/coinId 검증과 정리 로직 넣기. 그리고 저장 JSON 타입을 문서로 정리하고 배틀 API 테스트 취약점도 따로 정리하기.

## 해결하려고 한 문제

- 저장 데이터 오염
- 파일 저장 race condition
- 결과 적용 중복 가능성
- 오래된 클라이언트 상태 재사용
- 문서 부재

## 해결된 것

- 구현 변경 범위를 서버, 저장소, 클라이언트 상태, 문서까지 연결해서 정리했다.

## 아직 안 된 것

- 권한 이슈 없이 전체 테스트를 최종 확인하는 단계는 별도 검증이 필요하다.
