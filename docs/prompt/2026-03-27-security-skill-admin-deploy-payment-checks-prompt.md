# 보안 스킬에 관리자 분리, 배포 점검, 결제 시나리오 기준 추가 프롬프트

- 작성시각: 2026-03-27 11:23:15 +09:00
- 프롬프트:
  - “관리자 기능은 일반 유저 라우트와 코드베이스 레벨에서 완전히 분리해라. 미들웨어 하나로 막는 구조는 허용하지 않는다. IP 화이트리스트 옵션도 추가해라. 관리자 액션은 전부 로깅해라.”
  - 유저 생기기 전에 반드시 확인할 것
  - 배포 전 30초 체크

  - 배포 전 확인

  - 최근 추가된 운영 보안 기준을 전역 보안 스킬에 반영해서 이후 감사 시 빠뜨리지 않게 만드는 것

  - 관리자 기능 분리, 배포 전 점검, 결제/웹훅 시나리오, RLS/anon key/service_role/policy test 기준을 스킬에 반영함
