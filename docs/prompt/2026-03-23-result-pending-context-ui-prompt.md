# 결과 페이지 pending 문맥 UI 프롬프트

- 작성시각: 2026-03-23 20:19 KST
- 프롬프트 요약:
  - `http://127.0.0.1:3002/battle/bitcoin/result`에서 결과 확인 버튼을 누르면 버튼이 사라지고 카운트다운만 시작됨
  - 구조 설명 후, 결과 페이지가 비어 보이지 않게 2번 방식으로 수정 요청

## 해결하고자 한 문제

- 결과 페이지로 이동했는데 사용자가 여전히 `대기 화면`이라고 느끼지 못하고, 버튼만 사라진 것처럼 보이는 UX를 개선해야 했음
- 정산 전 result 화면에서도 현재 상태를 설명하는 문맥이 필요했음

## 해결된 것

- `settlementPending` 상태의 result 화면에 `MyPickSummary`와 설명 섹션을 추가했음
- 카운트다운 외에도 지금 진행 중인 단계와 저장된 발언 수를 보이게 했음
- 타입체크와 대상 파일 lint를 확인했음

## 해결되지 않은 것

- 버튼 라벨 문구 자체는 아직 그대로임
- 브라우저 상에서 waiting/result UX를 다시 전체 사이클로 검증하진 않았음
