  - `/login` 통합 로그인/회원가입 UX와 `/me` 요약 대시보드 중심 구조를 구현

## 해결하고자 한 문제

- planning 문서에는 인증 UX와 `/me` 대시보드 방향이 정리됐지만 실제 화면은 아직 이전 구조에 더 가까웠음
- `/login`은 로그인/가입 통합 의미와 에러 상태가 약했고, `/me`는 등급 노출과 첫 화면 정보 우선순위가 부족했음

## 해결된 것

- `/login`에 통합 로그인/회원가입 카피, provider별 CTA, callback 실패 안내를 추가함
- `/me`에 등급 카드와 압축된 최근 기록 중심 레이아웃을 반영함
- 헤더 비로그인 CTA를 `로그인/회원가입`으로 맞춤
- 로그인 화면 테스트를 추가해 OAuth 진입과 에러 노출을 검증할 준비를 함

## 해결되지 않은 것

- 실제 브라우저에서 디자인 체감 확인은 아직 남아 있음
- `/me` 상세 UI 자체를 더 크게 단순화할지는 추후 실사용 감각을 보고 더 다듬을 수 있음
