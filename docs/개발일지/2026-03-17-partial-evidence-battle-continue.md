# 부분 실데이터 실패 시 배틀 계속 진행

- 작성시각: 2026-03-17 17:02 KST
- 해결하고자 한 문제:
  - 실데이터 확보 실패가 생기면 배틀 전체가 중단되는 대신, 해당 팀원만 사과 메시지를 내고 나머지 캐릭터는 계속 진행해야 했다.

## 해결된 것

- `MarketData`의 보조 근거 필드를 nullable로 바꿨다.
- `fetchMarketData`는 CoinGecko 핵심 시세만 확보되면 보조 소스 실패를 null로 표시하고 배틀용 스냅샷 생성을 계속한다.
- 캐릭터별 필수 근거가 비면 `generateCharacterMessage`가 분석 대신 사과 메시지를 만든다.
- 사과 메시지는 `지금 의견을 제시하지 못해서 죄송합니다.` / `다음번에 하겠습니다.` 문구를 사용한다.
- 배틀 요약 카드도 null 지표는 숨기도록 조정했다.
- 관련 테스트를 추가해서 보조 소스 실패 시 partial 진행과 사과 메시지를 검증했다.

## 아직 안 된 것

- CoinGecko 핵심 시세 자체가 실패하면 배틀 시작은 여전히 불가능하다.
- 어떤 근거 소스가 비었는지 사용자에게 더 자세히 설명하는 UI는 아직 없다.
