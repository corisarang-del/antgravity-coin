# hover 프리뷰 복구와 비디오 폴백 정리

- 작성시각: 2026-03-15 14:47 KST
- 해결하고자 한 문제:
  - 첫 화면 캐릭터 라인업의 hover 프리뷰 오버레이를 다시 살려야 함
  - 업로드 이미지처럼 `preview.mp4` 안내문이나 깨진 파일 영역이 화면에 보이면 안 됨
- 해결된 것:
  - 랜딩 hover 프리뷰 오버레이를 다시 복구함
  - hover 프리뷰는 `/characters/previews/{id}.mp4` 영상을 우선 시도하도록 구성함
  - 영상이 없거나 로드 실패하면 파일명이나 플레이스홀더를 노출하지 않고 이미지 프리뷰로 자연스럽게 폴백되게 처리함
  - `pnpm lint`, `pnpm typecheck`를 다시 통과함
- 해결되지 않은 것:
  - 실제 mp4 파일은 아직 프로젝트에 없어서 현재는 이미지 폴백으로 동작함
