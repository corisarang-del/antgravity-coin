# 캐릭터 API 스키마 분리와 배틀 썸네일 연결

- 작성시각: 2026-03-15 12:10 KST
- 해결하고자 한 문제: 캐릭터 데이터를 나중에 DB나 외부 API로 바꾸기 쉽게 응답 스키마를 분리하고, 배틀 화면에서도 실제 캐릭터 이미지를 썸네일로 보여줘야 했어. 동시에 Aira와 Shade의 원본 이미지도 새 파일로 교체해야 했어.
- 해결된 것:
  - `Aira`, `Shade` 원본 PNG를 새 파일 기준으로 교체하고 대응 WebP를 다시 생성했어.
  - `src/features/characters/api.ts`에 캐릭터 API 응답 스키마와 변환 함수를 분리했어.
  - `src/app/api/characters/route.ts`를 추가해서 실제 캐릭터 응답을 받을 수 있게 했어.
  - `TeamBoard`, `SpeakerSpotlight`에 캐릭터 썸네일을 붙였어.
  - 관련 테스트와 운영 문서를 업데이트했어.
- 해결되지 않은 것:
  - DB나 외부 API 연동 자체는 아직 안 붙였어. 대신 응답 스키마와 변환 계층까지는 준비해뒀어.
