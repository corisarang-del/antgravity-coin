# memory.md 재독과 research.md 업데이트

- 작성시각: 2026-03-27 09:59 KST
- 해결하고자 한 문제:
  - `memory.md`를 부분 요약이 아니라 전체 운영 맥락으로 다시 이해하고, 그 이해를 다음 세션용 기준 문서인 `research.md`에 반영할 필요가 있었음

## 해결된 것

- `memory.md`를 처음부터 끝까지 다시 읽고 최근 체크포인트까지 포함한 핵심 판단을 정리했음
- 아래 관점을 이번에 다시 확정했음
  - 프로젝트 핵심 병목은 무료 모델 조합의 가용성과 tail latency임
  - 체감 속도 최적화의 기준은 첫 발언보다 `pick-ready` 시점임
  - 원격 반영 완료 범위와 local 미커밋 WIP를 구분해서 봐야 함
  - 다음 세션 시작 직후에는 구현보다 `judy` / `shade` 진행 여부 검증이 먼저임
- 위 이해를 `research.md` 상단 업데이트 섹션으로 추가했음

## 해결되지 않은 것

- local WIP 자체를 검증하거나 커밋한 것은 아님
- `judy`, `shade`의 `round=2` 이후 진행 여부는 다음 세션에서 실제 서버 재기동 후 다시 확인해야 함
- Supabase migration 실제 적용 여부와 production 관리자 권한 부여는 문서상 체크포인트로만 남아 있음
