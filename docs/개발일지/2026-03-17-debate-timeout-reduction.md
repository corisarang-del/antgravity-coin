# 토론 타임아웃 단축

- 작성시각: 2026-03-17 03:56 KST
- 해결하고자 한 문제:
  - 캐릭터 토론 생성이 오래 걸려 첫 응답과 전체 완료 시간이 늘어졌다.
  - 사용자 요청에 따라 컨텍스트나 프롬프트 구조는 유지하고, 타임아웃만 줄여서 더 빨리 fallback 되게 해야 했다.

## 해결된 것

- `src/shared/constants/characterModelRoutes.ts`에서 캐릭터별 `timeoutMs`를 전반적으로 낮췄다.
- balanced 계열은 `12000 -> 9000`, cheap 계열은 `10000/12000 -> 8000`, `flip`은 `12000 -> 9000`으로 조정했다.
- 프롬프트, 컨텍스트 전달, 호출 순서, fallback 로직은 그대로 유지했다.

## 아직 안 된 것

- 프롬프트 길이 축소나 병렬 호출 같은 구조적 속도 개선은 하지 않았다.
- 실제 체감 속도는 OpenRouter 응답 품질과 네트워크 상태에 따라 추가 확인이 필요하다.
