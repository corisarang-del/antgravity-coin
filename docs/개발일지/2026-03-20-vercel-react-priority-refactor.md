# Vercel React Priority Refactor

- 작성시각: 2026-03-20 17:45 KST
- 해결하고자 한 문제:
  - 배틀 SSE 시작 전에 불필요한 서버 워터폴이 있어서 첫 응답이 늦어지는 문제
  - 헤더 인증 상태를 클라이언트에서 다시 불러오면서 hydration 이후 추가 요청이 생기는 문제
  - 랜딩 페이지 대부분이 클라이언트 컴포넌트로 묶여 초기 번들이 과도하게 커지는 문제

## 해결된 것

- `src/app/api/battle/route.ts`에서 시장 데이터와 재사용 컨텍스트 로딩을 병렬 시작하도록 바꿔 첫 SSE 이벤트 지연을 줄였음
- 서버에서 현재 사용자 스냅샷을 미리 만들고 `AppHeader`에 주입하도록 바꿔, 헤더 기준으로는 초기 세션 표시를 서버 데이터로 바로 처리하도록 정리했음
- 랜딩 페이지를 서버 페이지로 재구성하고, 실제 인터랙션이 필요한 진입 오버레이와 캐릭터 프리뷰 레일만 클라이언트 아일랜드로 분리했음
- `pnpm typecheck`, `pnpm lint` 통과

## 해결되지 않은 것

- `useCurrentUserStore` 자체는 여전히 fallback용 클라이언트 fetch 경로를 유지하고 있어서, 초기 스냅샷을 주지 않는 다른 소비처가 생기면 추가 요청이 다시 발생할 수 있음
- 랜딩 페이지의 기존 한국어 문자열은 일부 파일 인코딩 이슈가 남아 있어 후속 정리가 필요할 수 있음
