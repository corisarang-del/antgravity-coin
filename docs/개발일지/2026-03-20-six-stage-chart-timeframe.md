# 6단계 차트형 타임프레임 전환

작성시각: 2026-03-20 11:15 KST

## 해결하려고 한 문제

- 배틀 시간 구간이 `24h`, `7d` 두 단계만 있어서 차트형 결과 경험이 단조로운 문제
- 결과 계산이 일간/주간 변화율만 바라봐서 5분, 30분, 1시간, 4시간 같은 세분화 구간을 표현할 수 없는 문제
- 선택 화면, 대기 화면, 결과 화면 문구가 실제 차트형 결과 브리핑 톤과 맞지 않는 문제
- 저장 스키마와 타입이 6단계 확장을 반영하지 못하는 문제

## 해결된 것

- `5m`, `30m`, `1h`, `4h`, `24h`, `7d` 6단계 공통 타임프레임 타입을 추가했다.
- 관련 도메인 스키마와 저장 타입이 모두 6단계를 수용하도록 확장했다.
- `resolveBattle`가 24시간 변화율과 보조 지표를 바탕으로 단기 구간용 파생 변화율을 계산하도록 바꿨다.
- `TimeframeSelector`를 6개 옵션 카드형 UI로 교체했다.
- pick, waiting, result 화면 문구를 실시간 차트형 결과 브리핑 톤으로 정리했다.
- `MyPickSummary`, `VerdictBanner`, `CountdownTimer`도 새 타임프레임 라벨을 반영하게 바꿨다.
- 전체 타입체크, 린트, 테스트를 통과했다.

## 아직 안 된 것

- 5분, 30분, 1시간, 4시간 결과는 실거래 히스토리 기반이 아니라 24시간 변화율 기반의 파생 값이다.
- 추후 실제 캔들 데이터 소스를 붙이면 현재 파생 계산 로직은 교체 대상이다.
