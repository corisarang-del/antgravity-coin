# vercel react 저우선순위 2차 스윕

- 작성시각: 2026-03-18 20:29 KST
- 해결하고자 한 문제:
  - 1차 스윕 이후에도 남아 있던 저우선순위 렌더링/JS 최적화 후보를 정리하고 싶었다.
  - 특히 외부 스토어 동기화, 이벤트 핸들러 안정화, 불필요한 배열 순회 축소를 끝까지 정리해야 했다.

## 해결된 것
- 최근 코인 훅을 `useSyncExternalStore` 기반으로 바꿔 동일 세션/탭 간 상태 동기화를 단순화했다.
- 승자 하이라이트의 `filter -> sort -> findIndex` 흐름을 줄여 중복 제거와 상위 메시지 선정을 더 단순하게 정리했다.
- 캐릭터 레벨 변화와 팀 카드 쪽 팀별 목록 계산을 상수화했다.
- Top coin 카드에 `content-visibility`를 추가해 렌더링 비용을 조금 더 줄였다.
- 카운트다운 타이머가 prop 변경 시 다시 동기화되고 0초일 때 불필요한 interval을 만들지 않도록 바꿨다.
- 캐릭터 상세 모달의 ESC 닫기 핸들러를 `useEffectEvent`로 정리했다.
- `pnpm lint`, `pnpm typecheck`를 다시 통과했다.

## 아직 해결 안 된 것
- low priority 규칙 중 실제 병목이 확인되지 않은 항목까지 억지로 넣지는 않았다.
- 예를 들어 대규모 리스트 전용 가상화, 추가 메모 분리, 더 공격적인 동적 import는 현재 구조에서 실익이 확인될 때 적용하는 쪽이 낫다.
