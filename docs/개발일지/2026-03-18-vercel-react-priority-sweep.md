# vercel react 우선순위 전반 최적화

- 작성시각: 2026-03-18 20:19 KST
- 해결하고자 한 문제:
  - Vercel React best practices 우선순위 기준으로 현재 Next.js/React 화면 전반에 있는 병목을 한 번에 줄이고 싶었다.
  - 중복 클라이언트 요청, 순차 API 호출, 초기 번들에 불필요하게 섞인 UI, 반복 계산을 같이 정리해야 했다.

## 해결된 것
- `useCurrentUser`를 공용 외부 스토어로 바꿔서 헤더와 결과 화면이 같은 세션 요청을 중복 호출하지 않게 했다.
- 결과 화면에서 배틀 적용 여부 조회와 기존 리포트 조회를 `Promise.all`로 병렬화했다.
- 홈 화면과 상단 코인 API가 같은 top coin 스냅샷 유즈케이스를 쓰도록 정리했다.
- 랜딩 모달을 분리하고 동적 import로 불러오도록 바꿔 초기 번들 부담을 줄였다.
- 검색창은 `useDeferredValue`와 `startTransition`을 써서 입력 반응성과 결과 갱신 우선순위를 분리했다.
- 선택 화면은 bull/bear 요약을 한 번의 순회로 계산하도록 바꿨다.
- 배틀 화면과 팀 보드는 작은 파생 계산을 더 단순하게 정리했다.
- `pnpm lint`, `pnpm typecheck`를 통과했다.

## 아직 해결 안 된 것
- 전 우선순위를 문자 그대로 모든 규칙에 맞게 전부 적용한 건 아니다.
- 데이터 직렬화 최소화, 추가 동적 import 후보, 고급 ref 패턴 같은 저우선순위 항목은 실제 병목이 확인된 뒤 이어서 다루는 게 맞다고 판단했다.
