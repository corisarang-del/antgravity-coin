# 캐릭터 성격과 토론 답변 정합성 보강

- 작성시각: 2026-03-18 22:09 KST
- 해결하고자 한 문제:
  - 캐릭터 성격, 선택 이유, 실제 토론 답변이 충분히 일치하지 않았고 답변이 차트 중심으로만 기우는 문제가 있었다.

## 해결된 것
- `GenerateDebateChunkInput`에 `role`, `personality`, `selectionReason`을 추가했다.
- 토론 시스템 프롬프트를 캐릭터별 역할/성격/말투 중심으로 다시 작성했다.
- 차트형 캐릭터가 아닌 경우 억지로 RSI/MACD만 반복하지 않도록 프롬프트 가드를 추가했다.
- 캐릭터별 fallback 요약/상세 문구도 역할에 맞게 전부 분리했다.
- `characters.ts`에서 깨진 캐릭터 설명 데이터를 정제된 한국어로 덮어써서 실제 UI/프롬프트 입력이 일치하도록 맞췄다.
- 관련 정적 검증과 테스트(`lint`, `typecheck`, 토론/캐릭터 관련 vitest`)를 통과했다.

## 아직 해결 안 된 것
- 실제 live API를 물린 실전 배틀에서 캐릭터별 답변 편차를 더 보려면 운영 로그를 한 번 더 보는 게 좋다.
- `characterDebateProfiles.ts` 안에 남아 있는 깨진 문자열 자체는 이번 턴에서 전면 정리하지 않았다.
