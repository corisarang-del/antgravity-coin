# Character Prompt And Evidence Split

- 작성시각: 2026-03-20 20:45 KST
- 해결하고자 한 문제:
  - 캐릭터 배틀 토론에서 역할 차이가 약하고, 거의 비슷한 차트 해설로 수렴하는 문제
  - 어떤 원소스를 읽고 말하는지 프롬프트 입력 단계에서 잘 드러나지 않는 문제

## 해결된 것

- 캐릭터 프롬프트 파일을 역할별 렌즈 중심으로 다시 구성했음
- 각 캐릭터마다 말투, 핵심 해석 축, 금지/주의 포인트를 더 강하게 분리했음
- 증거 문자열에 `원소스` 태그를 붙여 프롬프트 입력에서 출처가 드러나게 했음
- LLM 응답 캐시 키에 프롬프트 버전을 포함해 예전 결과가 재사용되지 않게 했음
- `pnpm lint`, `pnpm typecheck`, `pnpm test` 통과
- 브라우저에서 `ETH` 배틀을 직접 열어 첫 발언이 기존보다 차트 캐릭터 톤으로 더 명확해진 것 확인

## 해결되지 않은 것

- Judy, Clover, Vela처럼 차트 외 관점 캐릭터는 현재 데이터 모델상 숫자 요약 비중이 높아서, 완전히 차별화된 말투를 만들려면 뉴스 헤드라인/이벤트/원문 근거 같은 더 풍부한 입력이 필요함
- 현재 UI에는 원소스 태그를 직접 보여주지 않고, 프롬프트 입력에서만 사용함
