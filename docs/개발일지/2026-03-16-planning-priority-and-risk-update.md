# 개발일지 - planning 우선순위와 리스크 보강

- 작성시각: 2026-03-16 20:30:00 +09:00
- 해결하고자 한 문제:
  - `docs/planning` 문서에는 구현 단계와 후속 확장 순서는 있었지만, 지금 당장 해야 할 일과 나중에 해도 되는 일, 위험도가 높은 일을 한눈에 보는 우선순위 레이어가 약했음.
  - 특히 `06-tasks.md`는 단계별 계획은 있었지만, 현재 MVP 신뢰도 회복 작업과 고위험 구조 개편을 분리해서 읽기 어려웠음.
- 진행 내용:
  - `docs/planning/06-tasks.md`에 우선순위 A, 우선순위 B, 위험도 높은 작업 섹션을 추가함.
  - `docs/planning/01-prd.md`에 현재 우선 개발 원칙을 추가함.
  - `docs/planning/05-api-spec.md`, `docs/planning/04-data-model.md`에 정비 우선 항목과 데이터 리스크를 추가함.
  - `docs/planning/06-screens.md`에 화면별 우선 개선 포인트를 추가함.
- 해결된 것:
  - 지금 당장 해야 할 일, 나중에 해도 되는 일, 위험도가 높은 일이 문서상 명확히 분리됨.
  - 기존 완료 항목과 기존 Step 구조를 건드리지 않고 우선순위 레이어를 추가함.
  - PRD, API, 데이터 모델, 화면 문서가 같은 우선순위 관점을 공유하게 됨.
- 해결되지 않은 것:
  - 실제 코드 구현과 테스트는 이번 작업 범위에 포함하지 않았음.
  - 문서에 적은 고위험 작업의 세부 실행 절차와 단계별 롤백 기준은 아직 별도 문서화하지 않았음.
