# 배틀 결과 저장 안정성 강화

작성시각: 2026-03-20 00:00 KST

## 해결하려고 한 문제

- 배틀 결과 저장 전에 깨진 문자열과 영문 응답이 그대로 JSON에 남는 문제
- 파일 기반 저장소가 동시 write에 취약한 문제
- 결과 저장과 적용 마킹이 분리돼 idempotency가 약한 문제
- `battleSnapshot`, `userBattle`이 오래된 값으로 남아 다른 코인 화면에 섞이는 문제
- 저장 JSON 타입과 배틀 API 테스트 취약점이 문서화되지 않은 문제

## 해결된 것

- 결과 저장 전에 문자열 정합성 검증과 fallback 치환 로직을 추가했다.
- 파일 저장소 write를 저장 파일 단위 직렬화로 보호했다.
- `/api/battle/outcome` 안에서 결과 저장과 적용 마킹을 한 흐름으로 묶었다.
- `battleSnapshot`, `userBattle`에 TTL 및 `coinId` 검증을 넣었다.
- 저장 JSON 타입 문서와 배틀 API 테스트 취약점 문서를 추가했다.
- 관련 테스트를 추가했다.

## 아직 안 된 것

- 실제 데이터베이스 트랜잭션 수준의 원자성은 아니다. 파일 저장소 한계가 남아 있다.
- 배틀 API 전체 테스트를 실행해 동시성 회귀까지 검증하는 일은 환경 권한 이슈를 해결한 뒤 다시 확인이 필요하다.
